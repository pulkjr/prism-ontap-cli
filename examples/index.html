prism-ontap-cli

Prism.js syntax highlighting for NetApp ONTAP 9.x CLI

Volume show simple command + table output

Standard admin prompt, a two-word command, and a tabular result with column headers, separator, data rows, and a count line. 

cluster1::> volume show
  Vserver   Volume       Aggregate    State      Type       Size  Available Used%
  --------- ------------ ------------ ---------- ---- ---------- --------- -----
  vs0       vol0         aggr0        online     RW       1.0GB    972.7MB    5%
  vs1       vol1         aggr1        online     RW       2.0GB      1.8GB   10%
  vs1       vol2         aggr1        offline    RW       5.0GB      4.5GB   10%

Multiple parameter flags with different value types: a wildcard IP, pipe-separated state alternatives, a wildcard string, and a quoted timestamp.

"Fri Jul 17 11:38:08 2020" node1 ssh         10.60.250.79 volume create vol1 -aggregate test         Success -

Advanced privilege mode elevated prompt

The ::*> prompt is rendered in bold white to signal elevated privilege — visually distinct without conflicting with other token colors. 

cluster1::> set advanced
Warning: These advanced commands are potentially dangerous; use them only when
         directed to do so by NetApp personnel.
Do you want to continue? {y|n}: y

cluster1::*> storage aggregate show-space -aggregate aggr1
                                           Aggregate : aggr1
                                               State : online
                                 Total Physical Size : 8.17TB
                                  Used Physical Size : 6.22TB
                             Available Physical Size : 1.95TB

cluster1::*> set admin

Vserver context vserver prompt

SVM (vserver) administrator session — the prompt reflects the vserver name. Boolean state values, size values, and IP addresses each have distinct colors.

Network interface show IPv4 + IPv6 addresses

IPv4 and IPv6 addresses (including CIDR prefix) are highlighted in cyan. The hyphenated subcommand show-space is correctly tokenized as part of the command name, not as a parameter. 

cluster1::> network interface show -vserver vs1 -status-oper up

Vserver     Interface  Admin/Oper Address/Mask       Node          Port    Home
----------- ---------- ---------- ------------------ ------------- ------- ----
vs1         data_lif1    up/up    192.168.1.100/24   node1         e0a     true
vs1         data_lif2    up/up    192.168.1.101/24   node2         e0b     true

FCP adapter show WWPN / WWNN

World Wide Port Names (WWPN) and World Wide Node Names (WWNN) are highlighted in the same cyan as IP addresses. The 8-pair format is correctly distinguished from 6-pair MAC addresses. 

cluster1::> vserver fcp show
                                       Status
Vserver   Target Name                  Admin
--------- ---------------------------- ------
vs0       20:00:00:a0:98:0c:b0:eb      up
vs1       20:01:00:a0:98:0c:b0:eb      up
